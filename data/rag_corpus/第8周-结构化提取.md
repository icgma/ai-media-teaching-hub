## 一、导论：从“读”到“懂” (15分钟)

### 1.1 (5分钟) 回顾与挑战

*   **回顾：** 上节课我们学习了基础文本摘要，能快速获取文章大意。
*   **提问：** 但标准摘要够用吗？
    *   如果你的读者是专业投资者，他需要的是财报里的核心数据，而不是“公司前景广阔”的空话。
    *   如果你的任务是做竞品分析，你需要的是用户评论里的具体优缺点，而不是“这篇文章讲了用户反馈”。
*   **结论：** 基础摘要解决了“**是什么**”的问题，但无法高效回答“**所以呢？**”、“**细节是？**”、“**谁支持/谁反对？**”。我们需要从文本中**提取 (Extract)** 结构化的、有特定价值的信息。

### 1.2 (10分钟) 信息提取：现代记者的核心竞争力

*   **定义：** 信息提取 (Information Extraction, IE) 是指从非结构化的自然语言文本中，自动识别并抽取出结构化的信息片段，如实体、关系、事件等。
*   **对新闻业的价值：**
    *   **数据新闻：** 快速从成千上万份报告（如政府公报、企业财报、法律文书）中提取关键数据，制作可视化图表。
        *   *案例：* 《卫报》通过分析数万份泄露文件，揭示了全球范围内的税务问题。
    *   **调查报道：** 在海量信息中发现人物、组织之间的隐藏关系，找到调查线索。
    *   **舆情与受众分析：** 自动化分析社交媒体上的用户评论，快速了解公众对某一事件或产品的情感倾向、主要观点和提及的优缺点。
    *   **效率革命：** 将记者从重复、繁琐的“复制粘贴”工作中解放出来，专注于更高阶的判断、采访和写作。

## 二、为特定受众/风格生成摘要 (25分钟)

### 2.1 (10分钟) 理论：超越“一刀切”的摘要

*   **受众画像 (Persona) 的重要性：**
    *   **专业度：** 专家 vs. 公众 (术语使用、信息密度不同)
    *   **目的：** 决策者 vs. 研究者 (关注点：结论建议 vs. 方法论)
    *   **场景：** 移动端快速浏览 vs. 桌面端深度阅读 (篇幅、格式不同)
*   **风格 (Style) 的定义：**
    *   **平台风格：** 微博的短平快 vs. 微信公众号的深度图文 vs. 知乎的逻辑思辨。
    *   **语调 (Tone)：** 客观中立 (新闻)、热情洋溢 (营销)、严肃专业 (报告)、轻松幽默 (社交媒体)。
*   **AI实现原理：** 大语言模型 (LLM) 在训练时学习了海量的、带有不同风格标签的文本。通过**角色扮演 (Role-Playing)** 的提示词，我们可以激活它在特定“人格”下的语言模式。

### 2.2 (15分钟) 实战1：定制化摘要生成

*   **核心提示词技巧：** CRISPE框架中的 **C (角色)** 和 **S (风格)** 是关键。
*   **工具：** ChatGPT, Claude, 文心一言等。
*   **源文本 (示例)：**
    > *“A公司今日发布了其2025年第二季度财报。报告显示，公司营收达到500亿美元，同比增长15%，超出市场预期的480亿。其中，其创新的AI硬件部门‘星尘’系列产品销售额达100亿美元，成为新的增长引擎。但由于研发投入巨大，公司净利润为80亿美元，同比下降5%。CEO李明表示：‘我们对‘星尘’的长期潜力充满信心，暂时的利润压力是为未来投资。’财报发布后，公司股价盘后下跌3%。”*

*   **任务A：生成“微博版”摘要 (目标：快速传播，吸引眼球)**
    *   **提示词模板：**
        ```
        你是一名新浪财经的社交媒体编辑。请为以下财报新闻撰写一条微博，要求：
        1.  风格轻松、吸引人，使用一些热门标签 (emoji)。
        2.  字数严格控制在140字以内。
        3.  突出最核心的亮点和反差。
        4.  结尾引导用户讨论。

        [粘贴源文本]
        ```
    *   **预期输出示例：**
        > 业绩炸裂但股价跳水？🤔 A公司Q2财报来了！营收超预期达500亿，AI硬件“星尘”大卖100亿成新宠儿🚀。但高研发投入让净利下滑5%，股价盘后跌3%。CEO表示是为未来投资。大家怎么看这次的财报？是机会还是风险？#A公司财报# #AI硬件#

*   **任务B：生成“决策简报版”摘要 (目标：服务管理层，辅助决策)**
    *   **提示词模板：**
        ```
        你是一名专业的商业分析师，正在为公司管理层撰写一份关于竞争对手A公司的财报摘要。请基于以下新闻，用专业、客观、精炼的语言提炼核心信息，格式如下：

        **A公司Q2财报速览**
        - **核心业绩：** [营收、净利及其同比变化]
        - **业务亮点：** [表现最好的业务板块及其数据]
        - **潜在风险：** [财报中揭示的负面信息或挑战]
        - **高层观点：** [CEO的关键表态]
        - **市场反应：** [股价变化]

        [粘贴源文本]
        ```
    *   **预期输出示例：**
        > **A公司Q2财报速览**
        > - **核心业绩：** 营收500亿美元 (YoY +15%)，超市场预期；净利润80亿美元 (YoY -5%)。
        > - **业务亮点：** AI硬件“星尘”部门表现强劲，销售额100亿美元，已成为关键增长点。
        > - **潜在风险：** 利润端承压，高额研发投入影响短期盈利能力。
        > - **高层观点：** CEO李明强调对AI业务的长期战略投资，不在意短期利润波动。
        > - **市场反应：** 股价盘后下跌3%，显示市场对利润下滑的担忧。

## 三、关键信息提取 (KIE) (35分钟)

### 3.1 (10分钟) 理论：KIE的四大任务

1.  **命名实体识别 (NER - Named Entity Recognition):**
    *   **是什么：** 识别文本中具有特定意义的实体。
    *   **新闻应用：** 自动识别人名 (`PER`)、地名 (`LOC`)、组织机构名 (`ORG`)、时间 (`TIME`)等，构建新闻要素网络。
2.  **观点/情感提取 (Opinion/Sentiment Extraction):**
    *   **是什么：** 识别文本中表达的观点持有者、观点内容以及情感倾向（正面/负面/中性）。
    *   **新闻应用：** 快速分析某人对某事的看法；自动化处理海量评论，进行舆情监控。
3.  **关键数据提取 (Key Data Point Extraction):**
    *   **是什么：** 提取文本中的数值、百分比、金额等关键数据。
    *   **新闻应用：** 数据新闻的基础，从报告中快速抓取数据用于可视化。
4.  **关系提取 (Relation Extraction):**
    *   **是什么：** 识别实体之间的关系，如“A公司 **收购了** B公司”、“张三 **是** C公司的CEO”。
    *   **新闻应用：** 挖掘商业网络、人物关系，是调查报道的利器。

### 3.2 (25分钟) 实战2：从财报中提取结构化信息

*   **核心提示词技巧：** 明确要求AI“**提取 (Extract)**”、“**列出 (List)**”，并用**格式 (Format)**（如JSON、Markdown表格）来约束输出，这是让AI交付结构化数据的关键。
*   **源文本 (同上)**

*   **任务A：提取所有实体**
    *   **提示词：**
        ```
        请从以下文本中提取所有的命名实体，并按类别（公司、产品、人名、财务指标、时间）用Markdown表格格式化输出。

        [粘贴源文本]
        ```
    *   **预期输出：**
        | 类别 | 实体 |
        | :--- | :--- |
        | 公司 | A公司 |
        | 产品 | "星尘"系列 |
        | 人名 | 李明 |
        | 财务指标 | 营收、净利润、销售额、股价 |
        | 时间 | 2025年第二季度、盘后 |

*   **任务B：提取观点和情感倾向**
    *   **提示词：**
        ```
        请分析以下文本，提取其中包含的观点以及说话者的情感倾向。以JSON格式输出，包含`speaker` (说话人), `opinion` (观点内容), `sentiment` (情感倾向: '正面', '负面', '中性') 字段。

        [粘贴源文本]
        ```
    *   **预期输出：**
        ```json
        [
          {
            "speaker": "CEO李明",
            "opinion": "我们对‘星尘’的长期潜力充满信心，暂时的利润压力是为未来投资。",
            "sentiment": "正面"
          },
          {
            "speaker": "市场",
            "opinion": "财报发布后，公司股价盘后下跌3%",
            "sentiment": "负面"
          }
        ]
        ```

*   **任务C：提取所有关键数据**
    *   **提示词：**
        ```
        请提取以下文本中所有具体的财务数据，并以“指标：数值（同比变化）”的格式列表输出。

        [粘贴源文本]
        ```
    *   **预期输出：**
        - 营收：500亿美元（同比增长15%）
        - 市场预期营收：480亿美元
        - “星尘”产品销售额：100亿美元
        - 净利润：80亿美元（同比下降5%）
        - 股价：盘后下跌3%

## 四、非结构化文本的结构化 (30分钟)

### 4.1 (10分钟) 理论：从评论到洞察，从段落到表格

*   **场景1：用户评论分析**
    *   **痛点：** 上千条评论，如何快速知道大家在夸什么、骂什么？
    *   **AI方案：** 让AI阅读所有评论，自动分类、总结，并提取具有代表性的优缺点。
*   **场景2：信息结构化**
    *   **痛点：** 一大段描述性文字，信息点分散，难以对比和分析。
    *   **AI方案：** 定义好表格的列（字段），让AI自动填充，将非结构化文本转换为结构化表格。

### 4.2 (20分钟) 实战3：产品评论分析与文本转表格

*   **源文本 (模拟电商用户评论)：**
    > 评论1：“这款‘星尘’耳机音质真的绝了，低音很震撼！但续航有点拉胯，听4个小时就没电了。”
    > 评论2：“佩戴舒适度满分，戴一天耳朵也不疼。就是价格太贵了，性价比不高。”
    > 评论3：“连接速度超快，蓝牙秒连。不过，附送的充电线质量太差了，用两次就坏了。”

*   **任务A：从用户评论中提取优缺点和情感倾向**
    *   **提示词：**
        ```
        你是一名产品分析师。请阅读以下三条关于“星尘”耳机的用户评论，提取每条评论中提到的优点和缺点，并总结总体的情感倾向。请使用Markdown表格输出。

        | 评论ID | 优点 | 缺点 | 情感倾向 |
        | :--- | :--- | :--- | :--- |
        | 评论1 | ... | ... | ... |
        | 评论2 | ... | ... | ... |
        | 评论3 | ... | ... | ... |

        [粘贴三条评论]
        ```
    *   **预期输出：**
        | 评论ID | 优点 | 缺点 | 情感倾向 |
        | :--- | :--- | :--- | :--- |
        | 评论1 | 音质好，低音震撼 | 续航差 | 混合 |
        | 评论2 | 佩戴舒适 | 价格贵，性价比低 | 混合 |
        | 评论3 | 连接速度快 | 充电线质量差 | 混合 |

*   **任务B：将非结构化文本转换为结构化表格**
    *   **源文本 (一段描述)：**
        > “星尘耳机Pro版于2025年发布，售价1999元，采用蓝牙5.3技术，续航长达8小时。而标准版是2024年推出的，价格为1299元，使用蓝牙5.2，续航为5小时。”
    *   **提示词：**
        ```
        请将以下关于“星尘”耳机两个版本的描述性文本，转换为一个Markdown表格。表格需要包含以下列：`型号`, `发布年份`, `售价`, `蓝牙技术`, `续航时间`。

        [粘贴源文本]
        ```
    *   **预期输出：**
        | 型号 | 发布年份 | 售价 | 蓝牙技术 | 续航时间 |
        | :--- | :--- | :--- | :--- | :--- |
        | 星尘耳机Pro版 | 2025 | 1999元 | 蓝牙5.3 | 8小时 |
        | 星尘耳机标准版 | 2024 | 1299元 | 蓝牙5.2 | 5小时 |

## 五、总结与工作流整合 (15分钟)

### 5.1 (10分钟) 构建人机协作的高效信息处理工作流

1.  **明确目标 (人)：** 定义最终需要什么样式的洞察和数据。
2.  **AI批量初筛 (机)：** 使用AI工具对海量文本进行初步的摘要、分类和信息提取。
3.  **人工审核与校验 (人)：**
    *   **事实核查！** AI提取的数据、观点必须回溯原文进行验证。
    *   判断AI提取信息的**相关性**和**重要性**。
4.  **AI二次加工 (机)：** 将人工筛选后的信息，交由AI进行结构化转换（如生成表格）、风格改写或深度分析。
5.  **最终整合与呈现 (人)：** 记者/编辑将AI生成的结构化素材，整合成最终的报道、图表或报告，并加入人类独有的深度洞察和价值判断。

### 5.2 (5分钟) 总结、Q&A与作业布置

*   **核心要点总结：**
    *   高级摘要与提取的核心是**明确目标**和**约束格式**。
    *   **角色扮演**和**格式要求**是写好提示词的两大支柱。
    *   AI是强大的**助理**，但**事实核查**的责任永远在人类记者身上。
*   **课后作业：**
    1.  找一篇你感兴趣的上市公司最新季度的财报（PDF或网页版）。
    2.  **任务1：** 设计提示词，让AI为你生成一份面向“潜在股民”的财报解读（风格通俗，突出亮点和风险）。
    3.  **任务2：** 设计提示词，让AI从财报中提取一个关键业务板块过去三年的营收数据，并以Markdown表格形式输出。
    4.  提交你的两次提示词、AI的输出结果，以及你对结果的简要评价（是否准确？是否需要优化？）。
